# 운영자 가이드

## 상용 서비스로 운영하려면 추가로 필요한 것

1. 회원가입 / 로그인 정책
2. 사용자 데이터 삭제 기능
3. 개인정보 처리방침
4. 이용약관
5. 오류 로그 수집
6. 데이터 백업 정책
7. 결제 도입 시 결제/환불 정책
8. 세금/투자 정보 업데이트 정책

## 권장 운영 구조

초기에는 아래 구조를 권장합니다.

```text
프론트엔드: Vercel
DB/Auth: Supabase
저장 방식: local-first + cloud snapshot
백업: 사용자 직접 JSON 다운로드 + Supabase snapshot
```

## 업데이트 원칙

- 세법/투자 관련 수치는 코드에 고정하지 말고 설정값으로 관리
- 자동 분류 결과는 즉시 저장하지 말고 사용자 확인 후 저장
- 복원은 항상 비교 화면을 거치게 하기
- 대시보드 점수는 계산 근거를 함께 표시
